How it works

Passive sensing, zero friction

Asphalt runs silently in the background. Riders and drivers don't have to do anything — the SDK detects anomalies automatically and batches them for upload.

1

Sensor sampling

The SDK samples the accelerometer and gyroscope at 100 Hz, buffering data in a sliding window.

2

On-device detection

A lightweight algorithm looks for characteristic impulse signatures: sharp Z-axis peaks with gyro confirmation, vehicle-type-adjusted thresholds.

3

GPS tagging

Each detection is stamped with the device location. GPS switches to high-accuracy mode only when motion is detected, preserving battery.

4

Batch upload

Events are queued locally and uploaded as signed batches. Duplicate batches are idempotently ignored by the backend.

5

Spatial clustering

The backend runs DBSCAN every few minutes to group nearby events into anomaly clusters with a confidence score.

6

Open map

Clusters are served via a tile API and visualised on the live map — cross-validated by multiple vehicle types for higher confidence.

Vehicle types

Built for Indian roads

Different vehicles experience road anomalies very differently. Asphalt profiles each vehicle type with separate thresholds and uses cross-vehicle agreement as a confidence booster.

🛺

Three-wheelers (Autos)

Lower suspension stiffness means more baseline vibration. The SDK applies a three-wheeler filter to suppress false positives from engine vibration and tight turns.

🚗

Four-wheelers (Cars)

Standard detection profile. Reports from cars carry high confidence on their own, especially on urban roads.

🏍️

Two-wheelers Planned

Motorcycles and scooters have their own vibration profile. A two-wheeler filter is planned to reduce noise from road buzz and chassis resonance.

Privacy first

No personal data, ever

Asphalt was designed from day one to be privacy-preserving. Every architectural decision reflects this commitment.

🚫

No account required

There is no user registration. Sessions are ephemeral random UUIDs that are never associated with any identity.

📍

Coarse coordinates only

Event coordinates are truncated to ~11 m precision before storage. Exact device paths are never reconstructed.

📷

No camera, no microphone

Detection is purely sensor-based — accelerometer and gyroscope. No camera permission is ever requested.

🇮🇳

DPDP Act compliant

The demo app shows an explicit consent dialog before any data collection starts, aligned with India's Digital Personal Data Protection Act 2023.

🔓

Open source

The SDK, backend, and this website are fully open source. You can audit exactly what data is collected and how it is stored.

🗑️

No PII in database

The backend schema stores event_id, timestamp, truncated coordinates, intensity, and sensor summary. Nothing that can identify a person.

Built in the open

Asphalt is MIT-licensed and welcomes contributions. Whether you want to tune detection thresholds, add a new vehicle profile, or improve the map UI — all development happens publicly on GitHub.
